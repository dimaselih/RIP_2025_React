# Изображения для приложения

Эта папка содержит локальные изображения, которые не зависят от MinIO.

## Необходимые файлы

Добавьте следующие изображения в эту папку (`public/images/`):

### 1. `cart.png`
- **Назначение**: Иконка корзины
- **Размер**: Рекомендуется 26x26 пикселей (или больше, но пропорционально)
- **Формат**: PNG с прозрачным фоном
- **Использование**: Отображается в компоненте корзины на странице каталога услуг

### 2. `logo.png`
- **Назначение**: Логотип приложения
- **Размер**: Рекомендуется 80x50 пикселей (или больше, но пропорционально)
- **Формат**: PNG с прозрачным фоном (или без прозрачности)
- **Использование**: Отображается в навигационной панели (Navbar) в верхней части страницы

### 3. `default-service.png`
- **Назначение**: Изображение по умолчанию для карточек услуг
- **Размер**: Рекомендуется 300x200 пикселей (или больше, но пропорционально)
- **Формат**: PNG или JPG
- **Использование**: 
  - Используется в карточках услуг, если у услуги нет своего изображения
  - Используется как fallback, если изображение услуги не загрузилось
  - Отображается на странице деталей услуги, если изображение отсутствует

## Как это работает

Все изображения доступны через константы в `src/utils/imagePaths.ts`:

```typescript
IMAGES.CART          // /images/cart.png
IMAGES.LOGO          // /images/logo.png
IMAGES.DEFAULT_SERVICE // /images/default-service.png
```

React автоматически обрабатывает пути к файлам в папке `public/` через `process.env.PUBLIC_URL`.

## Проверка

После добавления изображений:
1. Убедитесь, что файлы имеют правильные названия (точно как указано выше)
2. Перезапустите dev-сервер (`npm start`)
3. Проверьте, что изображения отображаются в приложении
